Core additions: `FirestorePaths.singleBuildingId` (fixed building doc ID `"main"`, enabling a safe `exists()`-gated security rule); `firestore.rules`'s `isBootstrappingFirstAdmin`/`isBootstrappingAdminUserCreate`/`isBootstrappingAdminPrivateCreate` (each permanently false once the first sign-up succeeds; validated via `firebase deploy --only firestore:rules --dry-run` against the real project — compiled successfully, **not yet deployed**); `FirebaseAuthService.createUserWithEmailAndPassword`/`deleteCurrentUser` (the latter is compensating cleanup if the Firestore bootstrap batch fails after the auth account is created); `AuthRepositoryImpl.signUpAsAdmin`/`SignUpAsAdminUseCase`/`SignUpAsAdminRequested`. Fixed by requiring `kIsWeb` (Flutter Web only) in addition to the width check; no changes needed in `AuthScreen`/`SignUpScreen` themselves since their mobile layouts already had the wanted "logo + text at top, no side panel" design — the bug was purely which branch got selected. **(3) Splash screen added:** new `SplashScreen` (full-screen version of the web left brand panel, `AnimatedAppName` staggered letter reveal — the feature's third `StatefulWidget`) as the app's new initial route (`AppRoutes.splash`), gating on `AuthBloc`'s first non-`unknown` status (raced against a minimum display duration) before routing to Dashboard or Sign In — provider-agnostic auto-login reusing the existing `watchAuthState()`/Firebase session-persistence mechanism, no new auto-login logic built. `flutter analyze` clean (only pre-existing project-wide style infos) and `flutter test` shows the same 2 pre-existing unrelated failures as before this change. |
